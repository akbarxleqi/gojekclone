# iOS App Configuration for GoApp (KMP)

Since you are currently on Linux, you cannot run the iOS application yet. However, the structure has been prepared so that when you move to a Mac, you can run it with these steps:

## Prerequisites on Mac:
1.  **Xcode**: Install from the App Store.
2.  **Kotlin Multiplatform Plugin**: Install in Android Studio on Mac.
3.  **CocoaPods** (Optional): If you plan to use pods, but this setup uses a regular framework.

## How to Initialize the Xcode Project:
The best way to ensure the `.xcodeproj` is correctly configured for your specific environment is to:
1.  Open Android Studio on your Mac.
2.  The `iosApp` folder is already there with the Swift files.
3.  If the `.xcodeproj` is missing or not opening, you can create a new **SwiftUI** project in Xcode inside the `iosApp` directory and name it `iosApp`.
4.  Add the `ComposeApp.framework` (generated by the `:composeApp:embedAndSignAppleFrameworkForXcode` task) to your Xcode project.

## Current Setup:
- **MainViewController.kt**: Bridges Compose UI to iOS.
- **ContentView.swift**: Wraps the Compose UI in SwiftUI.
- **iOSApp.swift**: Entry point for the iOS application.

## Run via Command Line (on Mac):
Once on Mac, you can run:
```bash
./gradlew :composeApp:installIosSimulatorArm64Debug
```
(Assuming you are using an Apple Silicon Mac).
